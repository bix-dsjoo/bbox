# Bounding Box Labeler MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a runnable and testable Windows desktop Flutter MVP for local bounding-box labeling and COCO export.

**Architecture:** Start from a generated Flutter Windows project, then add pure domain modules first, followed by project persistence, import/detector/export services, viewer math and painting, and the workbench UI. Keep original image pixel coordinates in domain data and isolate display coordinate transforms in `lib/viewer`.

**Tech Stack:** Flutter 3.44.4, Dart 3.12.2, Material 3, `ChangeNotifier`, `file_picker`, `path`, and `image`.

---

## File Map

- Create generated Flutter files: `pubspec.yaml`, `analysis_options.yaml`, `lib/main.dart`, `test/widget_test.dart`, `windows/**`
- Create domain files:
  - `lib/annotation/models.dart`: project, image, box, label, statuses, JSON helpers
  - `lib/annotation/annotation_rules.dart`: validation, label rules, confirm rules
  - `test/annotation/annotation_rules_test.dart`
- Create project files:
  - `lib/project/project_store.dart`: UTF-8 save/load and schema checks
  - `test/project/project_store_test.dart`
- Create import/detector files:
  - `lib/image_import/image_scanner.dart`: supported file scan and dimensions
  - `lib/detector/detector.dart`: interface and deterministic dummy detector
  - `test/detector/dummy_detector_test.dart`
- Create viewer files:
  - `lib/viewer/viewport_transform.dart`: original/display transform math
  - `lib/viewer/box_overlay.dart`: painter and hit testing helpers
  - `test/viewer/viewport_transform_test.dart`
- Create export files:
  - `lib/export/coco_exporter.dart`: warning summary and COCO JSON
  - `test/export/coco_exporter_test.dart`
- Create UI files:
  - `lib/ui/app_controller.dart`: action-based state and autosave
  - `lib/ui/bbox_app.dart`: Material app root
  - `lib/ui/start_screen.dart`: create/open project entry
  - `lib/ui/workbench_screen.dart`: three-panel workbench
  - `lib/ui/image_list_panel.dart`
  - `lib/ui/viewer_panel.dart`
  - `lib/ui/box_list_panel.dart`
  - `lib/ui/label_panel.dart`
  - `lib/ui/export_dialog.dart`
  - `test/ui/workbench_widget_test.dart`
