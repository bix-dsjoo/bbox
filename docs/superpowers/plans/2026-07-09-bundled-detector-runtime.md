# Bundled Detector Runtime Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make automatic box proposals work on ordinary Windows PCs by packaging a CPU-only Python detector runtime with the app.

**Architecture:** The installed app first looks for `runtime/python/python.exe` next to `bbox_labeler.exe`, then falls back to environment overrides and developer-local Python paths. Packaging scripts create that runtime outside source control, and CMake/installer include it only when present. `train/` remains excluded.

**Tech Stack:** Flutter/Dart, Windows CMake install rules, Inno Setup, PowerShell, Python embeddable runtime, CPU PyTorch/torchvision, Ultralytics, OpenCV headless, NumPy.

## Global Constraints

- Do not bundle `train/` data.
- Do not commit large generated runtime files.
- Keep `FastSAM-s.pt` and detector scripts as product runtime assets.
- Prefer explicit app-directory paths over process working-directory paths.
- Preserve environment variable overrides for development and support.
- Workspace is not a Git repository, so commit steps are not available.
